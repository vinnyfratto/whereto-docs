# Changelog

All notable changes to WhereTo are recorded here. Format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/). Versioning follows the project
rule: `0.0.x` dev, `0.x.0` beta, `1.0.0` go-live, with `versionCode` incremented on
every change.

This file starts on 2026-07-05. History before this date lives in git commit messages
(`git log`). The weekly agent appends a new entry each week and at each version bump.

## [Unreleased]

### Added
- `docs/` engineering documentation system.
- Architecture Decision Records (`docs/decisions/`).
- Weekly review automation (`scripts/weekly-report-data.mjs`,
  `docs/automation/weekly-review-agent.md`, `/weekly-review` command).

## [0.3.43] - 2026-07-07

### Added
- Shared 4-tab top nav across all internal admin pages (Destination Images, Vibe
  Images, Vibes Engine, Vibe Engine Tester) via `scripts/lib/admin-nav.js`. The
  Vibe Engine Tester (search/scoring dashboard) is now a nav item instead of an
  orphan page.
- Bidirectional deep-links between destinations and vibes. A destination's vibe
  chips open that vibe in the Vibes Engine; a vibe's ranked-destination chips and
  the destination modal open that destination in Destination Images (new `?iata=`
  inbound resolver). Vibe Images links each vibe to its Vibes Engine entry.

## [0.3.37] - 2026-07-06

### Added
- Curated photo for every one of the 165 canonical vibes
  (`src/data/vibeCanonicalPhotos.ts`, generated by `scripts/build-vibe-photos.js` from
  Pexels). `getVibePhoto` now checks this map before the keyword guess, so the Discover
  planks, featured vibe cards, and results vibe filter all show on-topic images instead
  of fuzzy guesses or random picsum fallbacks (previously only 7 of 165 had a real
  image; 53 were random).

## [0.3.36] - 2026-07-06

### Fixed
- Discover vibe plank unchecked badge rendered as a flat dash. It used an absolute-fill
  circle inside a wrapper; switched to explicit 26x26 dimensions (the mood-tile
  pattern) so checked and unchecked are the identical round badge.

## [0.3.35] - 2026-07-06

### Fixed
- Discover vibe planks rendered broken (photos bleeding across rows, labels invisible,
  banded scrim): the all-absolute layout collapsed on device. Rebuilt with in-flow
  layout (fixed-height row, flex clip) and one uniform scrim.
- `getVibePhoto` now keyword-matches the curated photo map before falling back to
  random picsum images, so canonical labels ("Scuba & Reef Diving") get real photos.
  Pinned exact entries for Snorkeling, Sailing & Yacht Charters, Sea Caves & Blue
  Lagoons, and Stargazing & Dark Skies.

## [0.3.34] - 2026-07-06

### Changed
- Discover vibe planks redesigned as full-bleed photo strips (height 62 → 74): the
  vibe photo fills the plank behind a left-anchored scrim, white label, frosted check
  that fills rust on select, with a rust outline and a gentle photo zoom — matching
  the mood-tile and featured-card visual language.

_Note: versions 0.3.28–0.3.33 were bumped outside this changelog; see `git log`._

## [0.3.27] - 2026-07-05

### Fixed
- Discover Fine Tune now SCOPES the search to the chosen sub-region(s) instead of
  searching globally. Selections are tracked per sub-region (`fineBySub`), and the
  search fetches candidates via the sub-region-scoped `fetchCandidatesByCanonical` for
  each. Picking Ecuador & the Galapagos no longer returns Indonesia/Thailand/Brazil.

## [0.3.26] - 2026-07-05

### Fixed
- Discover selected mood tile now shows a full rust outline. The outline was a
  separate absolute-fill sibling that Android's `elevation` painted under the image
  (so only a bottom edge showed); moved the border onto the clipping container.

## [0.3.25] - 2026-07-05

### Fixed
- Removed `UIManager.setLayoutAnimationEnabledExperimental` calls (discover.tsx,
  CanonicalVibePicker.tsx) — they are no-ops on the New Architecture and only emitted
  warnings. LayoutAnimation still works via `configureNext`.

## [0.3.24] - 2026-07-05

### Fixed
- Flight cache no longer wedges AsyncStorage when the SQLite DB fills up
  (`SQLITE_FULL`). On a failed write it now purges expired entries (then all, if
  needed) and retries, so stale flight results can't accumulate and block other
  app storage.

## [0.3.23] - 2026-07-05

### Changed
- Discover mood tiles: selected state is now a full rust outline with the image zoomed
  15% inside the tile (tile size unchanged), replacing the underline-looking shadow.
- Discover trip-basics order is now Include → Starting Point → Budget → Dates →
  Travelers, and shows a "— N nights —" label under the dates (singular for one night),
  matching Wander with Vibes.
- Discover Fine Tune shows a "Vibes Specific to[ the] <sub-region>" header above the
  vibes, reusing the results hero's shared `articleFor` "the" logic.

## [0.3.22] - 2026-07-05

### Changed
- Discover step 1 mood tiles shortened so all six fit on screen (dashboard-style).
- Discover step 2 vibes are now single-column full-width "planks" (photo thumbnail +
  label + check), about six visible at load, instead of the 2-column photo tiles.

### Added
- Discover step 2 "Fine Tune" pill: swaps the generalized planks for a Region →
  Sub-Region → available-vibes browser (canonical vibe planks per sub-region). Multi
  select accumulates across region/sub-region changes. If any fine-tune vibes are
  selected they drive the search; otherwise the generalized mood-derived vibes do.

## [0.3.21] - 2026-07-05

### Added
- Results hero subtags now include a nights bullet (third, after From and budget)
  computed from the travel dates, with correct singular "1 night".

## [0.3.20] - 2026-07-05

### Changed
- Discover is now a 3-step walkthrough (mood → trip basics → vibes) that hands off to
  the STANDARD results screen instead of rendering its own reveal. Removed the
  sharpening hero image, the in-app priced cards, elimination round, postcard, and
  reveal refiners.
- Trip basics gains travel-date selection (same `DateRangeInput` as other workflows)
  and component stamps for Flights / Hotel / Rental Car (car = coming soon). Flights
  and Hotel default on and can be deselected (one must stay on); the Rental Car stamp
  is disabled.
- Results from Discover show the standard results page with a "Destinations to
  Discover" hero (Discover in rust) via `?from=discover`.

### Added
- `searchParams.includeFlights` / `includeHotels` (default true). `confirmSearch` now
  prices and budget-counts only the toggled-on components; every other workflow is
  unchanged because both default on.

### Fixed
- `confirmSearch` clears results and sets loading before its first await, so a caller
  that navigates then calls it never flashes the empty state.

## [0.3.19] - 2026-07-05

### Changed
- Discover step 2 is now a concrete "trip basics" step: Starting Point (the same
  airport picker as Wander with Vibes), Budget, and Travelers (passenger count). It
  replaces the qualitative "who's coming along" occasion/Trip Lens.
- Because origin, budget, and travelers are captured up front, the reveal now prices
  the picks immediately (real match score, in-budget, flights + hotel) instead of
  showing a rating-only preview first. Only a starting airport is required; dates
  default to the 1st of next month + 6 nights and are adjustable via the When refiner.
- Reveal refiners simplified to a When (dates) chip plus tap-to-edit chips for the
  starting point and budget (both jump back to the trip-basics step).

### Removed
- The occasion / Trip Lens step and its vibe boosts (superseded by trip basics).

## [0.3.18] - 2026-07-05

### Changed
- Discover: Budget moved onto the travelers step ("who is coming along"), using the
  same budget widget as Wander with Vibes (Total / Per Person toggle + slider). Removed
  the Budget refiner chip from the reveal; the occasion step no longer auto-advances.

## [0.3.17] - 2026-07-05

### Changed
- Discover reveal now renders the standard destination card (`SwipeableCard` from the
  results screen) with the full ScoreDisplay rating, swipe-left to remove (with
  next-best backfill), swipe-right to explore, and the Flight / Hotel / Car action bar.
- Discover prices its picks in place by reusing the store's `confirmSearch` pipeline
  (cheapest flight + best-fit hotel + finalized score), so the cards show real "around
  $X" and the Flight/Hotel sheets are live. The "When" refiner is now a real date range.
- Kept the Discover elimination round and postcard commit stamp on the standard cards.

### Added
- `setCurationSuggestions` store action so an external ranker (Discover) can feed the
  `confirmSearch` pipeline.
- Exported `SwipeableCard` from the results screen for reuse.

### Fixed
- `confirmSearch` now attaches the VibeMatch `ScoreDisplay` readout whenever a
  suggestion carries a `vibeScore`, instead of gating on the global feature flag, so
  Discover's engine scores render with the flag off.

## [0.3.16] - 2026-07-05

### Fixed
- Discover: the occasion (Trip Lens) was injecting its boost vibes as new stack
  entries and giving them tiebreak priority, so picking City + Food/Wine with a
  family occasion surfaced beach vibes. Occasion now only promotes vibes that a
  selected vacation type already carries; it never imports an unrelated vibe.

## [0.3.15] - 2026-07-05

### Added
- `discover_moods` Supabase table: the Discover step-1 vacation-type tiles (label,
  phrase, Pexels image, canonical vibe keys, entry animation, ordering) are now
  content-managed like `dashboard_cards`, with a bundled fallback in
  `src/data/discoverMoods.ts`.

### Changed
- Discover step 1 rebuilt: six most popular vacation types (Beach, City break,
  Mountains, Food & Wine, History & Culture, Adventure & Wildlife) with full-bleed
  Pexels photos and the dashboard's slide-in-from-outside entry animation.

### Fixed
- Discover grid tiles rendered with zero height on device (percent width +
  `aspectRatio` with absolutely-positioned children); tiles now use explicit pixel
  sizing.

## [0.3.14] - 2026-07-05

### Added
- **WhereTo: Discover** (`app/search/discover.tsx`): guided travel-agent search
  walkthrough. Sharpening hero image, 2x3 mood grid, skippable occasion (Trip Lens),
  pre-stacked canonical vibe grid, global VibeMatch reveal (top 6, no region gate),
  elimination round, When/From/Budget refiners, postcard commit stamp.
- `src/data/discoverMoods.ts` (moods/occasions to canonical vibes, stack builder) and
  `fetchCandidatesGlobalByCanonical` in `src/lib/vibeRankings.ts`.
- WhereTo: Direct placeholder screen (`app/search/direct.tsx`).
- Five `discover_*` PostHog events.
- Supabase: two dashboard cards (Discover, Direct) and their image tables
  (`Dashboard_Images-Discover`, `Dashboard_Images-Direct`).
- Tracker: `docs/DISCOVER_PROGRESS.md`.

### Changed
- Dashboard (`app/index.tsx`) rebuilt from four full-width split cards to a 2x3
  photo-tile grid (six tiles). Rotating images, icon bounce, and entry animations kept
  and generalized; fixed a stale three-slot `imageIndices` cast.
